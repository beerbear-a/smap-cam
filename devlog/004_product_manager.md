## 自己紹介

Mercari で検索・発見機能の PM を4年、その後 DeNA でモバイルゲームのグローバル展開を担当。
現在はフリーで「出荷できないアプリを出荷させる」専門。

**信条:** 「完璧なプロダクトは存在しない。出荷されないプロダクトは存在しないのと同じだ。」

チームが優秀なのは見ればわかる。iOS カメラは Tier-1 水準だし、Maya のフィルムルックは他に無い差別化要因。
ただ、今の状態で **App Store に並べることはできない**。理由を整理する。

### ❌ MVP 未実装（出荷ブロッカー）

| 機能 | PRD箇所 | 優先度 | 理由 |
|------|---------|--------|------|
| **動物タグ付け** (シャッター後3秒UI) | 3.1 | P0 | コアループが完成しない |

| **図鑑: レアリティ ★1〜4** | 3.4 | P1 | ゲーミフィケーションが機能しない |
| **マップ: 動物園ピン** | 3.5 | P1 | マップが地図のまま |
| **透かし自動付与** (シェア時) | 3.6 | P2 | あると SNS 拡散が強い |

### ⚠️ 差分サマリー

```
実装済み: 見た目 / フィルムルック / ナビゲーション
未実装  : データレイヤー全体（動物DB・チェックイン・タグ付け）
```

正直に言う。**UI は世界水準だが、アプリとして機能していない。**
カメラで撮れるが、何を撮ったかが記録されない。図鑑があるが埋まらない。
これはまだ「フィルム風カメラアプリ」であり「動物記録アプリ」ではない。

## MVP シップロードマップ

### Sprint 1（本日〜3日以内）: コアデータレイヤー 🏗️

**目標:** 「動物を撮る → 記録される」を完成させる

```
担当推奨: バックエンド / データモデル専門エンジニア

タスク:
1. AnimalDB (SQLite)
   - species テーブル (id, name_ja, name_en, rarity 1-4, silhouette_url)
   - encounter テーブル (id, photo_id, species_id, zoo_id, memo, created_at)
   - zoo テーブル (id, name, lat, lng, prefecture)

2. LocationService 拡張
   - 現在地から半径 5km 以内の zoo を検索
   - 手動選択フォールバック

3. CheckInNotifier
   - 動物園チェックイン状態管理
   - FilmSession に zoo_id を紐付け
```

### Sprint 2（3〜6日後）: タグ付けUX 🏷️

**目標:** シャッターを押したら3秒以内に動物を記録できる

```
担当推奨: UI/UXエンジニア（Maya or 新規召喚）

タスク:
1. TaggingSheet
   - 撮影後3秒タイムアウト付きボトムシート
   - 動物名テキスト入力 + サジェスト
   - スキップボタン

2. 図鑑: 未発見シルエット
   - species にデータがある動物はシルエット表示
   - 発見済みのみ写真表示

3. 図鑑: レアリティバッジ
   - ★ 1〜4 表示（species.rarity から）
```

### Sprint 3（6〜9日後）: マップ完成 + シェア 🗺️

**目標:** 「どこで会ったか」が地図に刻まれる

```
タスク:
1. MapScreen: 動物園ピン
   - 訪問済み zoo を MapboxAnnotation でピン
   - タップ → その zoo のセッション一覧

2. 透かし自動付与
   - 写真に @username · 動物園名 · ZootoCam を合成
   - image_gallery_saver で保存後、share_plus でシェア
```

### Sprint 4（9〜12日後）: QA & リリース準備 🚀

```
タスク:
1. 実機テスト (iOS/Android 各主要端末)
2. App Store Connect 申請準備
   - スクリーンショット (6.7" / 5.5" / iPad)
   - プライバシーポリシー (ローカルのみ・位置情報)
3. TestFlight ベータ配信
4. 初期動物データ投入 (50種程度)
```

## チームアサイン計画

| Sprint | 役割 | 推奨人材 |
|--------|------|---------|
| Sprint 1 | データモデル & DB | SQLite/Riverpod 専門エンジニア |
| Sprint 2-UI | タグ付け UX | Flutter UI スペシャリスト（Maya と相談） |
| Sprint 3 | マップ & シェア | Mapbox 経験者 |
| Sprint 4 | QA リード | iOS/Android QA |

**次に召喚すべきエンジニア:** データレイヤー専門家
動物DB設計 + LocationService + CheckIn のコアが全ての前提になっている。
ここがないと Sprint 2 以降が動かない。**今すぐ召喚を推奨。**

## 所感・小言

> iOS カメラは本物だ。AVFoundation をあそこまで書けるエンジニアは国内でも希少。
> Maya のフィルムルックは「なぜこう実装したか」の理由が全部書いてある。それが読めるチームは信頼できる。
> Kenji の `withOpacity` 修正、些細なことに見えるかもしれないが、そういう細部に気を配れる人間がいるチームは崩れない。
>
> ただ、**今のままではアプリとして出荷できない**ことを全員が認識してほしい。
> 美しい UI と機能するプロダクトは別物だ。
> Sprint 1 のデータレイヤーが全ての鍵。ここを死守する。
