# Current Issues — 2026-03-15

この文書は、2026-03-15 時点の実装状態と、まだ残っている課題を短く引き継ぐためのメモです。

## 1. いま保存しておきたい判断

- 当面は `camera-first` を優先する
- `AIで思い出を整理` は UI と土台は入ったが、正式機能としては一旦保留
- 理由は `規約`, `鍵管理`, `送信同意`, `説明責任` の負荷が重く、現段階ではカメラ体験の磨き込みを優先したほうがよいから
- 将来やるなら `自前API` 前提で再設計する
- `BYOK` や外部 AI の個人トークン入力も、いったん積極導入しない

## 2. 実装済みだが、まだ判断が必要なもの

### AI整理まわり

- 設定画面に `AI` セクションあり
- `AIログイン / 接続` 画面あり
- `AIで思い出を整理` の設定、現像後導線、Journal での下書き編集 UI あり
- ただし現在の生成は `ローカル仮実装` で、実際の backend API 接続は未実装
- 仕様検討ドキュメント:
  - `docs/AI_MEMORY_ASSIST_SPEC_2026-03-15.md`
  - `docs/AI_LIFELOG_NOTE_EXPORT_DESIGN_2026-03-15.md`

判断メモ:

- 現時点では「実験用にコードは残す」が、「正式導線として強く見せない」ほうが安全
- 次回着手時は、先に利用規約 / プライバシーポリシー / 同意文言を固める

## 3. カメラ周りの残課題

- 実機でプレビューの見え方をまだ継続確認したい
  - アスペクト比切替時のクロップ
  - フィルターの見え方
  - フィルム感の質感
- フィルムプレビューは以前より軽くしたが、まだ「フィルムカメラっぽさ」は詰め切れていない
- オーバーレイは矩形帯を減らして整理したが、今後また UI を触る場合は「上下が不自然に見えないか」を優先確認する
- iOS 26 実機向けに互換モードが複数入っているので、安定版 iOS で不要になったものは整理余地あり

## 4. アルバム周りの残課題

- アルバムは `写真を見る感じ` を戻す方向で改善済み
- フィルム一覧にはインデックスシートも混ぜてある
- `思い出ノート` セクションと AI 整理導線を追加済み

残課題:

- AI 機能を保留するなら、アルバムの `思い出ノート` 見せ方を弱めるか再整理する
- AIなしでも note に持っていきやすい `テンプレ書き出し` 導線は別案として有力
- ロール詳細の情報量が増えているので、実機で情報密度を再確認したい

## 5. モード切替と体験設計の残課題

- フィルム / インスタント切替は以前より分かりやすくしたが、まだ実機で迷わないか継続確認したい
- `CheckInScreen` はまだ zoo-first の名残が強い
- 将来的には `Start Roll` など camera-first の開始画面へ寄せる余地が大きい

## 6. 開発モード

- `ZOOTOCAM_DEV_MODE=true` で、開発中の制限をかなり緩める実装が入っている
- 実機確認時は便利だが、release の正式挙動確認時は define の有無に注意

## 7. 実機確認時の優先チェック

1. カメラ起動直後に落ちないか
2. フィルター / グレイン / プレビュー遅延が許容か
3. フィルム / インスタント切替が詰まらないか
4. アルバムからロール詳細 / メモ / 現像導線に自然に戻れるか
5. 設定の `AIログイン / 接続` が残っていて違和感がないか

## 8. すぐ手を付けるなら

優先順:

1. AI 機能を一旦どこまで見せるか整理する
2. カメラの実機 UX を再点検する
3. アルバムの note / メモ導線を AI なし前提でも成立させる
4. `CheckInScreen` を camera-first に寄せる
