# ZOOSMAP - Product Requirements Document (PRD)

> バージョン: 1.0
> 最終更新: 2026-03-13
> ステータス: 確定

---

## 1. プロダクトビジョン

### キャッチコピー
**「撮って、記録して、発信する。動物好きのためのカメラアプリ」**

### ミッション
動物との出会いを、消えない記憶として刻む。

### コアコンセプト
```
カメラロール（5,000枚）の中に埋もれる思い出
         ↓
「あのレッサーパンダ、いつどこで会ったっけ？」
         ↓
ZOOSMAPなら即座に引き出せる
種 × 動物園 × 日時 × 写真 × メモ
```

### 設計思想
- **Discovery Over Impression** — いいね数より、発見の感動
- **Quiet Observation** — リアルタイム共有より、静かな記録
- **Knowledge as Love** — 知ることで、動物を愛せるようになる

## 2. ターゲットユーザー

### プライマリー: 動物園SNS投稿者
```
属性
├── 年齢: 20〜35歳（女性中心）
├── 行動: 動物園・水族館に月1〜数回訪問
├── SNS: Twitter/X・Instagram に動物写真を投稿
└── 悩み: カメラロールが膨大で過去の写真を探せない

ペインポイント
├── 「あのカワウソ、どこの水族館だっけ？」
├── 透かしをきれいに入れる方法がわからない
├── 投稿のたびに場所・日時を手動で書く
└── 写真の盗用が心配
```

### セカンダリー: 一眼・ミラーレスカメラユーザー
```
属性
├── 年齢: 30〜50歳
├── 機材: Sony α / Canon R / Nikon Z など
├── 行動: カメラロール読み込みで後から整理
└── 需要: プロ品質のLUT + 著作権保護の透かし
```

### ターシャリー: 動物園を「もっと楽しみたい」ライト層
```
属性
├── 年齢: 20〜40歳・ファミリー層
├── 行動: 年数回の動物園訪問
└── 需要: 図鑑を埋める達成感・思い出の整理
```

## 3. コア機能定義

### MVP機能（リリース必須）

#### 3.1 カメラ
```
要件
├── 独自カメラ実装（iOS: AVFoundation / Android: CameraX）
├── LUTリアルタイムプレビュー
├── 元画像保存 + 加工版生成（非破壊編集）
├── シャッター → 動物タグ付け（3秒・スキップ可）
└── フラッシュ ON/OFF

無料LUT: 2種
有料LUT（¥370買い切り）: 全種
```

#### 3.1.1 フィルム/インスタント方針（体験の中核）
```
方針
├── フィルム: 撮影後は自動でロールに集約（勝手にまとめる）
├── インスタント: 後でロールにまとめる（手動）
└── 撮影体験に課金は入れない（撮影は全員同条件）

現像プリセット
├── Standard / Premium を全員に提供
└── Premiumは粒状感・周辺光量・色の深みを強化（保存時に焼き込み）
```

#### 3.2 チェックイン
```
要件
├── GPS で近くの動物園を自動検出
├── 手動選択（リストから）
├── 野生動物モード（任意の場所に記録）
└── チェックイン中はカメラと常時連携
```

#### 3.3 出会い記録
```
要件
├── 種 × 動物園 × 日時 × 写真 × メモ
├── 動物名タグ: テキスト入力 or リストから選択
├── メモ: 1行（任意）
└── あとから追記可能
```

#### 3.4 図鑑（ポケモン図鑑方式）
```
要件
├── 種 × 動物園 の組み合わせでコレクション
├── 発見済み: 自分の写真 + 日時 + 動物園
├── 未発見: シルエット表示（名前は見える）
├── レアリティ ★1〜4（国内飼育施設数で算出）
└── タブ切替: 特定の動物園 / 全国 / 全種
```

#### 3.5 マップ
```
要件
├── 自分の出会いが地図に刻まれる
├── 動物園単位でピン表示
├── タップ → その動物園での記録一覧
└── Mapbox 使用
```

#### 3.6 発信（透かし + シェア）
```
要件
├── 透かし自動付与（控えめデザイン）
│   └── @username · 動物園名 · ZOOSMAP
├── カスタム透かし（有料解放）
├── Twitter/X・Instagram・note へのシェア
└── share_plus 使用
```

### POST-MVP機能（将来フェーズ）

| 機能 | 優先度 | 理由 |
|------|--------|------|
| プッシュ通知（新着動物） | 高 | リテンション最重要 |
| カメラロール読み込み | 高 | 一眼ユーザー獲得 |
| 他ユーザーの記録を見る | 中 | SNS化 |
| Zooツーリズム（次の動物園提案） | 中 | 図鑑コンプへの動機 |
| EXIFアーカイブ（過去写真一括登録） | 中 | 初回体験向上 |
| AI種別提案 | 低 | iNaturalistと競合 |

## 4. 非機能要件

```
パフォーマンス
├── カメラ起動: 2秒以内
├── LUT切替: リアルタイム（60fps維持）
└── 写真保存: 3秒以内

オフライン
├── 全コア機能はオフラインで動作
├── 動物DB・図鑑・マップ: ローカルSQLite
└── ネット接続はシェア時のみ必要

プライバシー
├── 位置情報: 撮影時のみ取得・ローカル保存
├── リアルタイム位置共有: 実施しない
└── ユーザーデータ: 端末内完結（MVP）
```

---

## 5. 制約・やらないこと

```
やらないこと（意図的な非機能）
├── リアルタイム位置情報の共有
│   理由: 動物園内の混雑・プライバシー
├── 動物の個体識別・個体名管理
│   理由: DBメンテコストが高すぎる
├── コメント・いいね機能（MVP）
│   理由: SNS化は後フェーズ
└── 動画撮影
    理由: スコープを絞るため
```
