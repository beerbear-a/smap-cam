## 1. 思想の核心

noteから抽出した、このプロダクトの本質的な哲学。

### 「記録は未来の自分への贈り物」

> 「5年後、10年後、はたまたもっと先に、あの時こんな感じだったなと
>  思い返すために、思い出のアルバムを残し、記憶の補助線を引いておきたい」

> 「楽しい記憶を残すことは、未来の自分へ託す、小さな贈り物である」

アプリの目的は**今を楽しむこと**ではなく、**未来の自分に渡すこと**。

---

### 「不便さのなかでしか生まれない記録」

> 「フェスもまた不便を楽しむ場だ。思い通りにならないことだらけで、
>  それでも、むしろだからこそ、あの空気は特別になる」

> 「狙ったものには狙いが残るが、そこに記憶はあまり宿らないのかもしれない」

**制約が体験の質を上げる** — smap Camと完全に同じ構造。

---

### 「現代の消費文化への小さな抵抗」

> 「『映える』スポットや『瞬間』をあらかじめ把握し、
>  終わったそばから演出されたハイライトをストーリーにあげる」

> 「すぐに共有し、すぐに消費していく流れへの、ささやかな逆行」

> 「他人が作ったナラティブをまるで自分のものであるかのようにまとってしまいがちな
>  現代への、小さな抵抗」

FestivalCAMは**「映え」のためのアプリではない**。

---

### 「フェスは時代のモーメント」

> 「この年はこういう音楽が鳴っていて、こういう空気があって、自分はこうだった。
>  その積み重ねが、人生の年輪となっていく」

フェスは毎年開催される = **年輪を刻む場**。
FestivalCAMはその年輪の記録係。

---

### 「タイムカプセル性」

> 「現像を忘れていたことで、思いがけない再会になった」
> 「ズボラも意外と悪くない」

意図せず時間が経つことで価値が増す。
**忘れていた記憶が突然蘇る体験**こそが本質。

---

## 2. プロダクト定義

### キャッチコピー
**「音楽と過ごした空気を、未来の自分へ。」**

### ポジション
```
ZOOSMAP     = 動物との出会いを記録する図鑑カメラ
FestivalCAM = フェスの空気感を記録するタイムカプセルカメラ
```

### ターゲット
```
プライマリー
├── フェス・ライブに年1〜複数回行く人
├── 写ルンですをフェスに持って行く / 持って行きたい人
├── 昔のフェス写真がカメラロールに埋もれている人
└── 音楽と記憶に感情的な価値を感じる人（20〜40代）

セカンダリー
├── タイムテーブルを手書きで管理している人
└── 「あのフェスで誰を見た」を覚えていたい人
```

## 3. コア機能定義

### 3.1 フェスチェックイン
```
対応イベント
├── サマーソニック
├── フジロック
├── ロッキンオンジャパン
└── ... （主要フェス）+ カスタム作成

チェックイン情報
├── イベント名
├── 年（2016 / 2019 / 2022 ...）
└── 日程（Day1 / Day2 / Day3）
```

### 3.2 タイムテーブル
```
機能
├── 主要フェスの公式タイムテーブルをインポート
├── 「見たい」「見た」を一覧でチェック
├── 自分だけのタイムテーブルを作成・配信
└── 見たセットの時間・ステージが記録に紐づく

※ タイムテーブル配信機能 = 運営者自身が使いたい機能
```

### 3.3 カメラ
```
暗所・ライブ特化LUT
├── Stage Warm（ステージの暖かい照明）
├── Night Vivid（夜・暗所コントラスト強め）
├── Haze（霞・煙・野外フェス雰囲気）
├── Kodak400（写ルンです的なフィルム質感）← 核心
└── Flash（フラッシュ撮影風）

制約オプション（写ルンですモード）
├── 枚数制限: 27枚 / 36枚 / 無制限
├── 現像遅延: ON にすると「翌日まで見られない」
└── ※ smap Camの思想をオプション化
```

### 3.4 セット記録（アーティスト × フェス × 日時）
```
記録する情報
├── アーティスト名
├── フェス名・年・日
├── ステージ
├── 感想メモ（一言）
└── 写真

表示
├── 「見たアーティスト一覧」
├── 「フェス別アーカイブ」
└── 「サマソニ2016で見た人一覧」が即座に引き出せる
```

### 3.5 フェス図鑑
```
ZOOSMAPの図鑑 = 動物
FestivalCAMの図鑑 = アーティスト

アーティスト × フェス の組み合わせでコレクション

The 1975
├── サマーソニック 2019 ✅ 見た
├── フジロック 2023   ✅ 見た
└── フジロック 2026   ░░ 気になる

ライブ回数の多いアーティスト = レアリティ低
日本初登場 / 限定出演 = レアリティ高
```

### 3.6 タイムカプセル機能
```
noteの思想から生まれた機能

撮影後に「封印期間」を設定できる
├── すぐ見る（通常）
├── フェス終了後（翌日）
├── 1ヶ月後
└── 1年後（次のフェスシーズンに開封）

設定した期間が来ると通知
「あのサマソニの写真が現像されました」

= デジタルの写ルンです体験
```

### 3.7 発信
```
透かし
└── @username · SUMMER SONIC 2026 · FestivalCAM

シェア形式
└── 写真 + 「The 1975 @ サマソニ2026」
    + FestivalCAMロゴ（控えめ）

SNS連携
└── Twitter/X（フェス実況文化との親和性が高い）
```

## 4. ZOOSMAPとの比較

| | ZOOSMAP | FestivalCAM |
|--|---------|-------------|
| **記録の単位** | 動物 × 動物園 | アーティスト × フェス |
| **図鑑** | 種図鑑 | アーティスト図鑑 |
| **チェックイン** | 動物園（GPS） | フェス（手動選択） |
| **LUT** | 自然光・動物向け | 暗所・ステージ向け |
| **制約機能** | なし | 写ルンですモード（オプション） |
| **タイムカプセル** | なし | あり（核心機能） |
| **固有機能** | 野生動物モード | タイムテーブル配信 |
| **Core Engine** | **共通** | **共通** |

## 8. エコシステムにおける位置づけ

```
Core Engine
（カメラ + チェックイン + 記録 + 図鑑 + タイムカプセル + 発信）
         │
         ├── ZOOSMAP       動物 × 動物園
         ├── FestivalCAM   アーティスト × フェス  ← ここ
         ├── TravelCAM     スポット × 旅先
         └── SportCAM      選手・試合 × スタジアム
```

**FestivalCAMはエコシステムの第2弾として最適。**

理由:
- ZOOSMAPと思想が完全に一致（noteが証明）
- 運営者自身がヘビーユーザー = プロダクトの解像度が高い
- フェス × 写真 のSNSコミュニティが大きい
- タイムテーブル配信でB2Cではなくコミュニティ主導の成長が見込める
